# Фальшпол — руководство пользователя

Это актуальная инструкция по плагину фальшпола в pyRevit на 13.03.2026.

## Совместимость

- Проверено в Revit: 2024, 2026.
- Проверено в pyRevit: 6.1.0.x.
- Для других версий требуется отдельная проверка перед рабочим применением.

Панель рассчитана на работу через семейства, а не через Revit Parts:

- плитки размещаются как экземпляры семейства `ФП_Плитка`;
- лонжероны размещаются как экземпляры `ФП_Лонжерон`;
- стойки размещаются как экземпляры `ФП_Стойка`.

Главная идея плагина:

1. Сначала подготовить зону: база, периметр, сетка.
2. Потом найти лучшее смещение по реальной геометрии зоны.
3. Потом разместить подсистему и плитки уже по найденной раскладке.

## Что нужно до первого запуска

Перед началом работы проверьте следующее:

1. В проект загружены нужные семейства: `ФП_Плитка`, `ФП_Лонжерон`, `ФП_Стойка`.
2. Работа ведется на плане, а не на 3D-виде.
3. Есть исходное перекрытие, которое будет использоваться как зона фальшпола.
4. Параметры `FP_*` созданы в проекте и, при необходимости, в семействах.

Если проект новый, начните именно с кнопки `Параметры`.

## Структура панели

Текущая рабочая панель выглядит так:

1. `Параметры`
2. `1 Подготовка`
3. `2 Подобрать раскладку`
4. `3 Лонжероны`
5. `4 Стойки`
6. `5 Плитки`
7. `6 Очистка`
8. `7 Справка`

Служебные и устаревшие инструменты из основной панели убраны.

## Быстрый старт с нуля

Рекомендуемый порядок работы на новом проекте:

1. Откройте план нужного уровня.
2. В `Параметры -> Параметры проекта` создайте параметры проекта.
3. При необходимости в `Параметры -> Параметры семейств` создайте параметры для семейств.
4. В `1 Подготовка -> Перекрытие` выберите перекрытие, задайте базовую точку и шаг плитки.
5. В `1 Подготовка -> Периметр` постройте точный периметр зоны.
6. В `1 Подготовка -> Сетка` постройте модульную сетку.
7. В `2 Подобрать раскладку` найдите лучшее смещение по фактической геометрии.
8. В `3 Лонжероны -> Разместить` постройте верхние и нижние лонжероны.
9. В `4 Стойки -> Разместить` расставьте стойки по нижним лонжеронам.
10. В `5 Плитки -> Разместить` разместите плитки по точным ячейкам.

Если нужно перепостроить зону заново, используйте `6 Очистка`, а потом снова идите по шагам сверху вниз.

## Что делает каждая кнопка

### Параметры

Внутри три команды:

- `Параметры проекта` — добавляет все обязательные `FP_*` параметры в проект;
- `Параметры семейств` — добавляет `FP_*` параметры в семейства фальшпола;
- `Очистить параметры` — удаляет параметры и их привязки.

Используйте этот блок в начале настройки или при полном пересборе структуры параметров.

### 1 Подготовка

Это выпадающий блок из трех шагов.

#### 1 Перекрытие

Что делает:

- выбирает исходное перекрытие;
- записывает базовую точку;
- записывает шаги плитки;
- сбрасывает стартовое смещение.

Что заполняет:

- `FP_Шаг_X`, `FP_Шаг_Y`;
- `FP_База_X`, `FP_База_Y`, `FP_База_Z`;
- `FP_Смещение_X`, `FP_Смещение_Y`.

Примечание по базовой точке:

- базовая точка записывается параметрами `FP_База_X/Y/Z`;
- визуально она помечается крестом на шаге `3 Сетка`;
- ссылка на маркер хранится в `FP_ID_МаркераБазы`.

#### 2 Периметр

Что делает:

- строит модельные линии по верхней грани зоны;
- записывает их как источник точной геометрии.

Важно:

- именно этот контур используется точным алгоритмом раскладки;
- если зона изменилась, периметр нужно перестроить заново.

Что заполняет:

- `FP_ID_ЛинийКонтура`.

#### 3 Сетка

Что делает:

- строит модульную сетку по заданному шагу;
- подрезает линии по периметру, если он уже построен;
- обновляет линии сетки при повторном запуске;
- рисует маркер базовой точки стилем `ФП_База`.

Что заполняет:

- `FP_ID_ЛинийСетки`.
- `FP_ID_МаркераБазы`.

### 2 Подобрать раскладку

Это ключевая кнопка плагина.

Она не просто двигает сетку, а реально оценивает раскладку плитки внутри фактического периметра:

- перебирает варианты смещения X/Y;
- считает точные ячейки по контуру;
- сравнивает варианты по качеству раскладки;
- выбирает лучшее смещение;
- записывает найденный результат в параметры зоны;
- сразу перерисовывает сетку с новым смещением.

Как работает подбор смещения внутри:

1. Считывается фактический контур из `FP_ID_ЛинийКонтура`.
2. Генерируются кандидаты смещения по X и Y в пределах шага.
3. Для каждого кандидата выполняется точный расчет ячеек внутри контура.
4. Оцениваются метрики качества раскладки (полные, подрезки, сложные, узкие фрагменты, зоны риска у отверстий/колонн).
5. Выполняется ранжирование вариантов по приоритетам качества.
6. Лучший вариант уточняется и округляется до практичного шага.
7. Итог записывается в `FP_Смещение_X/Y` и применяется к сетке.

Алгоритм штрафует плохие варианты по нескольким критериям:

- слишком узкие подрезки;
- микрофрагменты;
- сложные ячейки;
- проблемные зоны у отверстий и колонн;
- нежелательные участки для подсистемы.

После этой команды именно найденное смещение считается рабочим для дальнейшего размещения.

### 3 Лонжероны

Внутри две команды:

- `Разместить` — строит верхние и нижние лонжероны;
- `Удалить` — удаляет размещенные лонжероны.

Текущая логика:

- верхние идут по линиям сетки;
- нижние строятся перпендикулярно верхним;
- учитываются стыковочные выступы и максимальная длина;
- лишние участки подрезаются по контуру.

Что заполняет:

- `FP_ID_Лонжеронов_Верх`;
- `FP_ID_Лонжеронов_Низ`.

### 4 Стойки

Внутри две команды:

- `Разместить` — расставляет стойки по нижним лонжеронам;
- `Удалить` — удаляет размещенные стойки.

Текущая логика:

- стойки ставятся по концам и промежуточным точкам нижних лонжеронов;
- высота стойки рассчитывается автоматически;
- прямоугольная стойка поворачивается по оси нижних лонжеронов.

Если нижние лонжероны идут по Y, стойки автоматически разворачиваются на 90 градусов.

Что заполняет:

- `FP_ID_Стоек`.

### 5 Плитки

Внутри три команды:

- `Разместить` — размещает плитки;
- `Вентиляция` — помечает выбранные плитки как вентилируемые;
- `Удалить` — удаляет размещенные плитки.

Текущая логика размещения:

- все плитки ставятся по центру ячейки;
- полные, простые подрезки и сложные формы размещаются через единый точный алгоритм;
- и простые, и сложные вырезы делаются одним механизмом выреза (void);
- вырез считается как `cell_rect - clipped_polygon`;
- смещение выреза задается от левого и нижнего края ячейки, а не от центра.

Классификация плиток:

- `Полная`;
- `Подрезка`;
- `Сложная`.

Что заполняет:

- `FP_ID_Плиток`;
- параметры экземпляров плитки вроде `FP_Тип_Плитки`, `FP_Подрезка_X`, `FP_Подрезка_Y`, `FP_Вырез_*`.

### 6 Очистка

Удаляет элементы, созданные плагином:

- плитки;
- стойки;
- лонжероны;
- линии сетки;
- линии контура.

Используйте эту кнопку, если хотите полностью пересобрать зону без ручной чистки.

### 7 Справка

Открывает это руководство.

Назначение кнопки:

- быстро объяснить порядок работы новому пользователю;
- напомнить текущую логику плагина;
- собрать в одном месте шаги, ошибки и правила перепостроения.

## Ключевая логика, о которой важно помнить

### 1. Семейства, а не Parts

Плагин работает через семейства, а не через нарезку перекрытия на Parts.

Это сделано потому что:

- плитка остается BIM-элементом;
- проще специфицировать типы и количество;
- одинаковая логика используется для плиток, лонжеронов и стоек.

### 2. Источник истины

Источником истины являются:

- параметры исходного перекрытия;
- точный контур в `FP_ID_ЛинийКонтура`;
- актуальное смещение в `FP_Смещение_X/Y`.

Если результат выглядит странно, обычно проблема в одном из этих трех источников.

### 3. Два слоя модели

Плагин работает в двух слоях:

- модульная сетка — для конструктивной логики и подсистемы;
- точная раскладка — для точного анализа плитки по реальному контуру.

Это не конкурирующие, а дополняющие друг друга уровни.

## Частые ошибки и что делать

### Команда не работает

Проверьте:

- открыт ли план;
- выбрано ли правильное перекрытие;
- созданы ли параметры `FP_*`.

### Раскладка выглядит плохо

Сделайте по порядку:

1. Перестройте `Периметр`.
2. Перестройте `Сетку`.
3. Снова запустите `2 Подобрать раскладку`.

### Стойки не ставятся

Причина почти всегда одна:

- нижние лонжероны еще не размещены;
- или они были удалены после расчета.

Сначала заново выполните `3 Лонжероны -> Разместить`.

### Плитки размещены неактуально

Если менялись:

- шаг;
- база;
- периметр;
- смещение;
- лонжероны или стойки,

то лучше выполнить:

1. `5 Плитки -> Удалить`.
2. При необходимости заново `2 Подобрать раскладку`.
3. Снова `5 Плитки -> Разместить`.

### Нужно начать заново

Самый надежный путь:

1. `6 Очистка`.
2. `1 Подготовка -> Перекрытие`.
3. `1 Подготовка -> Периметр`.
4. `1 Подготовка -> Сетка`.
5. `2 Подобрать раскладку`.
6. `3 Лонжероны`.
7. `4 Стойки`.
8. `5 Плитки`.

## Основные параметры зоны

Для нормальной работы инструмента критичны:

- `FP_Шаг_X`, `FP_Шаг_Y`;
- `FP_Смещение_X`, `FP_Смещение_Y`;
- `FP_База_X`, `FP_База_Y`, `FP_База_Z`;
- `FP_ID_ЛинийКонтура`;
- `FP_ID_ЛинийСетки`;
- `FP_ID_Плиток`;
- `FP_ID_Лонжеронов_Верх`, `FP_ID_Лонжеронов_Низ`;
- `FP_ID_Стоек`.

## Что сейчас считается актуальным состоянием плагина

На текущий момент подтверждено:

- простые и сложные подрезки плиток работают через единый механизм выреза (void);
- плитки размещаются по центру ячейки;
- контур после перерисовки поднимается поверх сетки;
- стойки автоматически разворачиваются по оси нижних лонжеронов;
- служебные и устаревшие команды убраны из основной панели.

## Если вы новый пользователь

Если сомневаетесь, что делать дальше, придерживайтесь простого правила:

1. Сначала подготовьте геометрию.
2. Потом подберите раскладку.
3. Потом стройте каркас.
4. Потом размещайте плитки.

И только если результат уже не устраивает, запускайте `6 Очистка` и повторяйте последовательность шагов заново.
