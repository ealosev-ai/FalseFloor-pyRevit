# FalseFloor.extension

pyRevit-расширение с инструментом фальшпола.

## Совместимость и проверенные версии

- Проверено в Revit: 2024, 2026.
- Проверено в pyRevit: 6.1.0.x.
- На других версиях работа возможна, но требует отдельной проверки.

## Быстрый старт для тестера

1. Скачайте репозиторий или zip-архив.
2. Добавьте путь к папке `FalseFloor.extension` в pyRevit:
    - pyRevit -> Extensions -> Add Folder
3. Выполните Reload pyRevit.
4. Загрузите семейства в проект Revit из папки `FalseFloor.extension/Families`:
    - `ФП_Плитка.rfa`
    - `ФП_Лонжерон.rfa`
    - `ФП_Стойка.rfa`
5. Откройте вкладку `Фальшпол`.
6. Для пошаговой инструкции используйте кнопку `7 Справка`.

## Changelog: Оптимизатор раскладки

### Улучшения алгоритма подбора смещения (`find_best_shift`)

1. **REFINE_TOP_N 3 → 5** — фаза уточнения теперь работает по 5 лучшим
   кандидатам вместо 3, что расширяет зону поиска оптимума.

2. **Полный штраф по внешнему контуру** — `outer_roundness_penalty` учитывает
   все 4 ребра bbox (min_x, max_x, min_y, max_y), а не только min_x и min_y.

3. **Дедупликация snap-кандидатов с точностью 1 мм** — вместо `round(v, 12)`
   используется округление до 1 мм (`round(internal_to_mm(v))`), что убирает
   тысячи практически одинаковых кандидатов и ускоряет расчёт.

4. **Удалён мёртвый код `score`** — числовой `score = (...)` не использовался
   при сортировке (использовался `rank_key`). Убран из `evaluate_shift_exact`
   и из UI-вывода.

5. **Фаза 3: top-3 кандидата** — аналитические дельты `_cut_round_deltas`
   применяются к 3 лучшим кандидатам, а не только к лучшему, что повышает
   шанс найти вариант с кратными подрезками.

6. **Взвешенный near_edge_penalty** — вместо целочисленного счётчика
   `near_edge_count` добавлен дробный штраф `clearance / distance`:
   чем ближе линия сетки к ребру колонны, тем сильнее штраф.
   Целочисленный `near_edge_count` сохранён как первичный ранг,
   а `near_edge_penalty` выступает уточняющим тайбрекером.

7. **Cut spread penalty** — новый тайбрекер: стандартное отклонение размеров
   viable подрезок. Предпочитает однородные подрезки (например, 150 + 150
   лучше, чем 100 + 200).

### Визуальные улучшения сетки

- Линии сетки — василькoвый синий (100, 149, 237).
- Линии сетки вблизи колонн — красный (255, 80, 80).
- Контур перекрытия — зелёный (0, 255, 0).
- Порог «вблизи колонны» синхронизирован с шириной профиля лонжерона
  (`ФП_Ширина_Профиля` из семейства `ФП_Лонжерон`).
